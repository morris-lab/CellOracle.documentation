Overview¶

This notebook describes how to check distribution range of simulated value generated by celloracle in silico TF perturbation. Please read our paper to learn more about the CellOracle algorithm.

Notebook file¶

Notebook file is available at CellOracle GitHub. https://github.com/morris-lab/CellOracle/blob/master/docs/notebooks/05_simulation/misc/Check_distribution_of_simulation_value.ipynb

Aim of this analysis¶

In CellOracle gene perturbation simulation, celloracle simulates small shifts of downstream gene expression values upon TF perturbation. This simulation is NOT intended to predict out-of-distribution, and the range of simulated values should not differ largely from the original gene expression distribution range.

The range of simulated values is automatically evaluated when the simulation is run, and a warning is displayed if the simulated value range deviate considerably from the original gene expression distribution.

Here we also introduce another sanity check function that evaluates and visualizes the distribution of simulated and original gene expression values.

In this note, we will illustrate the use of this function using the results of the Spi1 KO simulation as an example of a CellOracle simulation. It is assumed that you have already used the tutorial notebook describing a typical CellOracle in a sillico TF perturbation simulation. If not, please see that first.

Steps¶

  1. We will perform TF perturbation simulation as usual.

  2. After simulation, we will examine the distribution of simulation values by comparing them with the original gene expression value distribution.

This comparison will clarify how the simulated values distribute. We see similar distribution between original and simulation values in typical celloracle simulation results because CellOracle intends to simulate only small initial shifts upon TF perturbation. If you see a large discrepancy or large out-of-distribution simulation, please confirm you are using appropriate parameters in your simulation.

1. Perform gene perturbation Simulation.¶

[5]:

# Load the tutorial oracle object and Links.
oracle = co.data.load_tutorial_oracle_object()
links = co.data.load_tutorial_links_object()

# Preprocessing
links.filter_links()
oracle.get_cluster_specific_TFdict_from_Links(links_object=links)
oracle.fit_GRN_for_simulation(alpha=10,
                              use_cluster_specific_TFdict=True)

# Perform in silico gene KO simulation.
oracle.simulate_shift(perturb_condition={"Spi1": 0.0},
                      n_propagation=3)

2. Check distribution of simulated values¶

After the simulation above, celloracle automatically evaluate simulated value range and display warnings if the simulation value range is largely different from original distribution range.

We can visualize the distribution as a histogram using a function below. This function returns top n genes that showed the bigest difference.

  • As default, it shows top 4 genes hat showed the bigest difference. We can change the number of genes by changing n_genes.

  • We can save plots by specifying folder name using save.

[6]:

oracle.evaluate_and_plot_simulation_value_distribution(n_genes=4,
                                                       save=None)

In this example above, the top four most differentially expressed genes in the Spi1 KO simulation are shown. Since Celloracle is intended to simulate only small initial shifts upon TF perturbation, the distribution range of simulated gene expression values should normally be similar to the distribution range of the original values.

If the distributions of simulated and original values differ significantly, please check if the input data and hyperparameters are appropriate.

3. Optional step: Clip out-of-distribution value¶

Also, you can consider clipping the out-of-distribution prediction value if you see significant out-of-distribution simulation values. Please perform simulation with clip_delta_X=True. In that case, all simulation values outside of the orignal gene expression range will be clipped.

[7]:

# Perform in silico gene KO simulation.
oracle.simulate_shift(perturb_condition={"Spi1": 0.0},
                      n_propagation=3,
                      clip_delta_X=True)
